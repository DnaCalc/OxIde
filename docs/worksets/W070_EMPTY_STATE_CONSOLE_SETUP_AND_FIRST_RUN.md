# W070 Empty State Console Setup And First Run

Status: `planned`
Date: 2026-04-02

## 1. Purpose
This workset delivers the two early-use surfaces already identified as
strategically important:
1. the empty/welcome state,
2. console capability testing and setup guidance.

## 2. Why This Is Early
These surfaces matter before the full IDE is complete because they shape:
1. first-run perception,
2. user onboarding,
3. terminal-environment validation,
4. whether users blame OxIde for terminal problems it could have explained.

## 3. Closure Condition
This workset closes when:
1. OxIde has a deliberate empty/welcome state,
2. OxIde has a first-class console diagnostics/setup surface,
3. Windows guidance is explicit,
4. the rebuilt shell treats these as product features, not debug leftovers.

## 4. Initial Epic Lanes
1. empty state
2. console diagnostics surface
3. Windows terminal/font guidance
4. first-run affordances

# OxIde Design Mockup from Figma (targeting TUI but presented as web mockup)

## Figma Design

I've created a comprehensive high-fidelity TUI design exploration for OxIde. The mockup showcases five distinct states demonstrating the IDE's visual language, spatial hierarchy, and interaction design:

## Design System Features:

Premium Terminal Palette: Dark background (#0a0e14) with disciplined accent colors (cyan #39bae6 for primary actions, subtle grays for hierarchy)
Strong Panel Identity: Clear borders, titled regions, and distinct visual zones for each surface
Grid-Based Composition: Honest terminal layout with explicit focus indicators and clean linework
Rich Information Density: Status indicators, diagnostics, symbols, and runtime state visible without clutter

## Five Screen States:

Empty/First-Run: Welcome screen with quick actions, terminal capability checks, and clear onboarding paths
Main Editing: Project explorer, code editor with syntax highlighting, inspector panel with diagnostics and symbols, and problems panel
Semantic-Rich: Shows hover documentation, immediate panel REPL, and references view demonstrating OxVba's language services integration
Build/Run: Live runtime state, execution output, build logs, and process controls showcasing IDE orchestration
Command Palette: Transient overlay showing unified command system with VBA-compatible mnemonics, search, and keyboard shortcuts

## Key Design Principles Applied:

IDE-first (not editor-first) with persistent project awareness
Buffer/view/layout model visible throughout
Keyboard-first with visible shortcuts and focus indicators
Terminal-honest constraints (no fake pixels, grid-based)
Calm, instrument-grade aesthetic with intentional color use
Strong semantic surfaces (diagnostics, symbols, immediate panel) as first-class citizens
The design demonstrates how OxIde differentiates from generic terminal editors by making project state, semantic analysis, and execution orchestration feel native to the TUI shell.

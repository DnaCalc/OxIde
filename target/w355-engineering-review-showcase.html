oxide-oxvba calls load_basproj and compile_project directly, then projects counts/diagnostics into OxIde view models.

 105  pub fn compile_build_check(
 106      workspace_path: impl AsRef<Path>,
 107  ) -> Result<CompileBuildCheckView, CompileBuildAdapterError> {
 108      let workspace_path = workspace_path.as_ref();
 109      let loaded = load_basproj(workspace_path).map_err(|source| CompileBuildAdapterError::LoadProject {
 110          path: workspace_path.to_path_buf(),
 111          message: source.to_string(),
 112      })?;
 113      let project_name = loaded.manifest.project_name.clone();
 114      let loaded_summary = compile_options_profile_from_loaded(workspace_path, &loaded);
 115  
 116      match compile_project(&loaded.manifest) {
 117          Ok(compiled) => Ok(CompileBuildCheckView {
 118              command_status: CompileCommandStatus::Succeeded,
 119              project_path: workspace_path.display().to_string(),
 120              project_name,
 121              diagnostics: Vec::new(),
 122              compiled_summary: Some(CompiledProjectSummary {
 123                  instruction_count: compiled.bytecode.instructions.len(),
 124                  slot_count: compiled.bytecode.slot_count,
 125                  user_slot_count: compiled.bytecode.user_slot_count,
 126                  procedure_count: compiled.procedure_runtime_metadata.len(),
 127                  host_export_count: compiled.host_exports.len(),
 128                  reference_visible_export_count: compiled.reference_visible_exports.len(),
 129                  event_binding_count: compiled.event_dispatch_bindings.len(),
 130                  dynamic_object_count: compiled.project_dynamic_objects.len(),
 131                  com_withevents_route_count: compiled.project_com_withevents_routes.len(),
 132                  rewritten_source_length: compiled.rewritten_source.len(),
 133              }),
 134              loaded_summary,
 135          }),
 136          Err(source) => Ok(CompileBuildCheckView {
 137              command_status: CompileCommandStatus::Failed,
 138              project_path: workspace_path.display().to_string(),
 139              project_name,
 140              diagnostics: vec![CompileDiagnostic {
 141                  phase: String::from("compile"),
 142                  code: source.code().to_string(),
 143                  message: source.to_string(),
 144                  source: String::from("oxvba_compiler::compile_project"),
 145              }],
 146              compiled_summary: None,
 147              loaded_summary,
 148          }),
 149      }
 150  }

Date: 2026-05-09

- Explicitly blocked by named OxVba wasm-safety gaps in the current frozen OxVba snapshot.

WASM probes run:
1. cargo check -p oxvba-compiler --target wasm32-unknown-unknown
   Result: failed.
   Primary gap: oxvba-compiler pulls oxvba-com. oxvba-com fails on wasm32 because resolve_typelib_identity_from_prog_id is Windows-gated and native dynamic-library functions are unavailable: libc::dlopen, libc::dlsym, libc::dlerror, RTLD_NOW, RTLD_LOCAL.

2. cargo check -p oxvba-web-host --target wasm32-unknown-unknown
   Result: failed.
   Primary gap: oxvba-web-host dependency graph reaches oxvba-com and native/JIT memory-protection dependency region. wasm32 check fails in oxvba-com and region.

3. cargo check -p oxvba-project --target wasm32-unknown-unknown
   Result: failed.
   Primary gap: project loading path reaches oxvba-com and region dependency paths for wasm32.

Command/client verification run:
- node apps/dna-oxide/scripts/verify-command-client.mjs
- Result: passed.

OxIde code behavior:
- apps/dna-oxide/src/command-client.js defines BROWSER_WASM_COMPILE_PROFILE.
- Browser fixture command client returns browserWasmCompileCheckUnavailableResponse for dna_oxide_build_check / compile.check.
- Browser fixture command client returns browserWasmCompileOptionsUnavailableResponse for dna_oxide_get_compile_options / compile.options.
